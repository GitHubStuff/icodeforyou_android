## 24.0.4-01 — Launcher Icons (from PNG)

Generate the app icons from the icon `.png` using Android Studio's Image Asset Studio. No hand-written XML in this step — Asset Studio generates the adaptive-icon layers and both `mipmap-anydpi-v26` files the spec requires (0.4), replacing the wizard's default robot icons.

### Prerequisite

The icon `.png` should be square, ideally 1024×1024 or larger, with a transparent background if the artwork isn't full-bleed. Know its location on disk.

### Steps

1. **Project view:** right-click `CatsCarsCoins/app` → **New → Image Asset**.
2. **Icon type:** `Launcher Icons (Adaptive and Legacy)`. **Name:** `ic_launcher` (default — keep it; the manifest already references it).
3. **Foreground Layer** tab: **Asset Type = Image**, **Path** = the icon `.png`. Use the **Resize** slider so the artwork fits inside the safe-zone circle preview (adaptive icons crop aggressively — keep important detail inside the circle).
4. **Background Layer** tab: **Asset Type = Color** and pick a color that suits the artwork (or **Image** if a background layer image exists).
5. **Next** → review the file list → **Finish**.

### What gets generated (replacing wizard defaults)

- `res/mipmap-anydpi-v26/ic_launcher.xml` and `ic_launcher_round.xml` — the spec-required adaptive icon definitions
- `res/drawable/ic_launcher_foreground.xml` (or `mipmap-*/ic_launcher_foreground.png` per density, when the source is a PNG)
- `res/values/ic_launcher_background.xml` (background color, if Color was chosen)
- Legacy density `mipmap-*dpi/` PNGs for pre-API-26 surfaces (Play listing, some launchers)

No manifest change — `@mipmap/ic_launcher` / `@mipmap/ic_launcher_round` references are already in place from the wizard.

### Verify

**Build → Make Project**, then check the file list above exists under `app/src/main/res/`. Icon renders in the preview pane of either `mipmap-anydpi-v26` XML.
