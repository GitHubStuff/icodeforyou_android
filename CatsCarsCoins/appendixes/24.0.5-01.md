## 24.0.5-01 — `di/AppModules.kt` (Koin aggregator stub)

Create new file `app/src/main/java/com/icodeforyou/catscarscoins/di/AppModules.kt` (**Project view:** right-click `java/com.icodeforyou.catscarscoins` → New → Package → `di`, then New → Kotlin Class/File → File → `AppModules`). Full content:

```kotlin
// di/AppModules.kt
package com.icodeforyou.catscarscoins.di

import org.koin.core.module.Module

/**
 * Aggregates every feature's Koin module (spec §3: one module per feature,
 * AppModules aggregates). Empty at standup (spec 0.5); each feature phase
 * appends its module here — the only DI file MyApp ever needs to know about.
 */
val appModules: List<Module> = emptyList()
```

**Why:** spec 0.5 requires `MyApp` to call `startKoin { modules(appModules) }` against an empty stub. Landing the stub first means the `MyApp` step compiles the moment it's written — no forward references, every step green.
