## 24.0.3-02 — Top-Level `build.gradle.kts`

Replace the entire contents of the **project-root** `build.gradle.kts` (**Project view:** `CatsCarsCoins/build.gradle.kts` — not `app/`) with the file below. **Do not sync** — 24.0.3-02 through 24.0.3-04 land together, with one sync at the end of -04.

**Why:** AGP 9's built-in Kotlin puts the Kotlin Gradle plugin on the build classpath at a version the plugin resolver reports as "unknown", so KGP-family sub-plugins (serialization, KSP) cannot be requested *with* a version — that produces "already on the classpath with an unknown version". Each sub-plugin is its own artifact, so every one `:app` uses is declared here at a known version; modules then apply them with no version and they resolve from this classpath. Versions are read from the catalog, keeping it the single source of truth.

### The file

```kotlin
// build.gradle.kts (project root)
// Top-level build file where you can add configuration options common to all sub-projects/modules.

// AGP 9 built-in Kotlin: KGP is on the build classpath at a version the plugin
// resolver can't see, so KGP-family sub-plugins cannot be requested WITH a
// version. Each sub-plugin is its OWN artifact — every one :app uses is
// declared here at a KNOWN version. Modules apply them with no version.
buildscript {
    repositories {
        google()
        mavenCentral()
    }
    dependencies {
        classpath("org.jetbrains.kotlin:kotlin-gradle-plugin:${libs.versions.kotlin.get()}")
        classpath("org.jetbrains.kotlin:kotlin-serialization:${libs.versions.kotlin.get()}")
        classpath("com.google.devtools.ksp:com.google.devtools.ksp.gradle.plugin:${libs.versions.ksp.get()}")
    }
}

plugins {
    alias(libs.plugins.android.application) apply false
    alias(libs.plugins.kotlin.compose) apply false
}
```
