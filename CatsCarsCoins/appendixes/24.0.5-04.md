## 24.0.5-04 — `ui/theme/Theme.kt` (`AppTheme`, dynamic color OFF)

Replace the entire contents of `app/src/main/java/com/icodeforyou/catscarscoins/ui/theme/Theme.kt` with the file below. `Color.kt` and `Type.kt` in the same package stay wizard-as-is — `Purple80`/`Purple40` etc. and `Typography` are referenced unchanged.

**Changes from the wizard file:**

- `CatsCarsCoinsTheme` → **`AppTheme`** — the spec's name (0.5: `AppTheme` with light/dark palettes). One rename now beats carrying a wrong name into every screen.
- **Dynamic color removed entirely** — spec 0.5: "dynamic off." Not defaulted-off: deleted, so no caller can flip it back on and the `Build.VERSION` branch and both `dynamic*ColorScheme` imports go with it.
- `darkTheme: Boolean = isSystemInDarkTheme()` stays — this is what makes `theme = system` follow the live device switch (paired with the §4 rule: no `uiMode` in `configChanges`).

### The file

```kotlin
// ui/theme/Theme.kt
package com.icodeforyou.catscarscoins.ui.theme

import androidx.compose.foundation.isSystemInDarkTheme
import androidx.compose.material3.MaterialTheme
import androidx.compose.material3.darkColorScheme
import androidx.compose.material3.lightColorScheme
import androidx.compose.runtime.Composable

private val DarkColorScheme = darkColorScheme(
    primary = Purple80,
    secondary = PurpleGrey80,
    tertiary = Pink80,
)

private val LightColorScheme = lightColorScheme(
    primary = Purple40,
    secondary = PurpleGrey40,
    tertiary = Pink40,
)

/**
 * App-wide Material3 theme (spec 0.5): explicit light/dark palettes,
 * dynamic color deliberately unsupported — palettes stay deterministic
 * across devices, which a reference blueprint requires.
 */
@Composable
fun AppTheme(
    darkTheme: Boolean = isSystemInDarkTheme(),
    content: @Composable () -> Unit,
) {
    MaterialTheme(
        colorScheme = if (darkTheme) DarkColorScheme else LightColorScheme,
        typography = Typography,
        content = content,
    )
}
```

**Expected fallout:** `MainActivity` still calls `CatsCarsCoinsTheme` — the build goes red until the `MainActivity` step lands next. This is the one deliberate exception to step-green discipline: the rename and its only caller are two halves of one change, split only to honor one-file-per-step.
