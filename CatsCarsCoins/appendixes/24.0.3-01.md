## 24.0.3-01 — Verify `settings.gradle.kts`

This step is verification only — the wizard normally generates a spec-conformant file, so there may be **no edits and no commit**. Only edit if a checklist item below is missing.

File location — **Android view:** Gradle Scripts → `settings.gradle.kts (Project Settings)`. **Project view:** `CatsCarsCoins/settings.gradle.kts`. Same file.

### Checklist

- ☐ `pluginManagement { repositories { ... } }` contains `google`, `mavenCentral()`, `gradlePluginPortal()`
- ☐ `dependencyResolutionManagement { repositories { ... } }` contains `google()`, `mavenCentral()`
- ☐ `rootProject.name = "CatsCarsCoins"`
- ☐ `include(":app")`

Wizard extras that are correct and must be **kept** if present:

- `content { includeGroupByRegex(...) }` filters on the `google` repo — resolution-speed optimization (only queries Google's repo for Google-owned groups).
- `org.gradle.toolchains.foojay-resolver-convention` plugin — toolchain auto-provisioner; lets Gradle download a JDK 17 on any machine where the daemon-JVM pin can't find one.
- `repositoriesMode.set(RepositoriesMode.FAIL_ON_PROJECT_REPOS)` — enforces single-source-of-repos; no module can declare its own repository.

### What the file should look like

```kotlin
pluginManagement {
    repositories {
        google {
            content {
                includeGroupByRegex("com\\.android.*")
                includeGroupByRegex("com\\.google.*")
                includeGroupByRegex("androidx.*")
            }
        }
        mavenCentral()
        gradlePluginPortal()
    }
}
plugins {
    id("org.gradle.toolchains.foojay-resolver-convention") version "1.0.0"
}
dependencyResolutionManagement {
    repositoriesMode.set(RepositoriesMode.FAIL_ON_PROJECT_REPOS)
    repositories {
        google()
        mavenCentral()
    }
}

rootProject.name = "CatsCarsCoins"
include(":app")
```
