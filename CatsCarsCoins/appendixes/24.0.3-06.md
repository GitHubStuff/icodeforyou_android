## 24.0.3-06 — Top-Level `build.gradle.kts` (KSP classpath coordinates) + Sync

Replace the entire contents of the **project-root** `build.gradle.kts` (**Project view:** `CatsCarsCoins/build.gradle.kts` — not `app/`) with the file below.

Then run the single sync for 24.0.3-05 → -06: **File → Sync Project with Gradle Files.** Green sync with **no** "kotlin.sourceSets DSL" warnings = 24.0.3 config proven.

**Why:** only change from 24.0.3-02 is the KSP classpath line — the artifact is now `com.google.devtools.ksp:symbol-processing-gradle-plugin`, the exact coordinates the AGP 9 release notes prescribe for pinning a higher KSP on the classpath (the previous plugin-marker coordinates worked but are the indirect form). The version it resolves, 2.3.3, comes from the catalog (24.0.3-05).

### The file

```kotlin
// build.gradle.kts (project root)
// Top-level build file where you can add configuration options common to all sub-projects/modules.

// AGP 9 built-in Kotlin: KGP is on the build classpath at a version the plugin
// resolver can't see, so KGP-family sub-plugins cannot be requested WITH a
// version. Each sub-plugin is its OWN artifact — every one :app uses is
// declared here at a KNOWN version. Modules apply them with no version.
buildscript {
    repositories {
        google()
        mavenCentral()
    }
    dependencies {
        classpath("org.jetbrains.kotlin:kotlin-gradle-plugin:${libs.versions.kotlin.get()}")
        classpath("org.jetbrains.kotlin:kotlin-serialization:${libs.versions.kotlin.get()}")
        // AGP-documented coordinates for pinning a higher KSP than AGP's default
        classpath("com.google.devtools.ksp:symbol-processing-gradle-plugin:${libs.versions.ksp.get()}")
    }
}

plugins {
    alias(libs.plugins.android.application) apply false
    alias(libs.plugins.kotlin.compose) apply false
}
```
