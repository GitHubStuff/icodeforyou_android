## 24.0.2-01 — Version Catalog (`gradle/libs.versions.toml`)

Replaced the wizard-generated version catalog with the complete file below, then **File → Sync Project with Gradle Files**. Sync note: catalog entries only resolve when a build script references them, so sync stays green here — each pin is proven as it gets wired into `app/build.gradle.kts` in later steps. Commit is its own step.

**What was done:** every added version was verified against current release pages, then corrected against the IDE's resolver after the first sync. Wizard entries were preserved verbatim — the wizard already generated the spec's three pins (agp 9.2.1, kotlin 2.2.10, composeBom 2026.02.01). Every addition is marked with an `# ── ADDED:` banner stating its purpose and spec reference.

**Decisions and flags baked into this file:**

- **Room 3.0.0** (Boss ruling): new coordinates `androidx.room3:*`, KSP-only, coroutines-first. The `androidx.room3` Gradle plugin with `schemaDirectory()` replaces the spec's `ksp { arg("room.schemaLocation", ...) }` — spec §2 needs that row updated. `@Fts4` support in Room 3 still to be verified before the first entity is written.
- **`adaptive = 1.3.0-rc01`** — the only non-stable pin. The `adaptive-navigation3` artifact (list-detail SceneStrategy, spec §5) does not exist in the stable 1.2.0 line; 1.3.0-rc01 is the only line carrying it. Revisit when 1.3.0 stable lands.
- **`kotlinxSerialization = 1.9.0`, not 1.10/1.11** — newer lines are built on newer Kotlin than this project's 2.2.10; the IDE's "newer version available" nag doesn't know about compiler/runtime pairing. Bump together with `kotlin`.
- **`ksp = 2.2.10-2.0.2`** — exact Kotlin-2.2.10-matched release (KSP only went Kotlin-version-independent from its 2.3.0 line).
- **Nav3 1.1.4** (current stable), **Koin BOM 4.2.2**, **OkHttp BOM 5.4.0**, **Retrofit 3.0.0** (+ first-party `converter-kotlinx-serialization`), **Coil 3.5.0**, **DataStore 1.2.1**, **splashscreen 1.2.0**, test libs (coroutines-test 1.11.0, Turbine 1.2.1, MockK 1.14.11, Kover 0.9.8).

**Post-sync corrections (correction #5, from IDE resolver output):** four originally-researched pins were stale and were bumped to the resolver's current stables — Kover 0.9.1→0.9.8, Koin BOM 4.2.1→4.2.2, coroutines-test 1.10.2→1.11.0, MockK 1.14.5→1.14.11. The file below is the corrected version.

**Warnings that remain after sync — expected, by policy:**

- "Newer version available" for the Kotlin plugins and Compose BOM: spec pins; they move by spec revision, never the IDE banner.
- "Newer version available" for kotlinx-serialization-json: compiler/runtime pairing pin (see above).
- "Dependency/plugin alias not used in build scripts" for every ADDED entry: burns down as each one is wired into `app/build.gradle.kts` in later steps.
