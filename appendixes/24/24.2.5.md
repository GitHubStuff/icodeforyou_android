## Design

- **Version 1 = coins only.** Cats (FTS4) and Cars entities arrive in their
  phases with version bumps; the tracked `schemas/` directory records each
  schema as history (spec 0.7's payoff starts now).
- **`exportSchema = true` explicit** ‚Äî the schema export is a spec
  requirement, not a default to rely on.
- **`db/` package**, feature-neutral: the database aggregates entities from
  every feature package; it belongs to none of them.
- Project wiring verified against source: plugin `androidx.room3`,
  `room3 { schemaDirectory("$projectDir/schemas") }`, `room3-runtime` +
  `ksp(room3-compiler)` ‚Äî all already correct; no build changes this step.

## Open item this gate adjudicates ‚Äî `@ConstructedBy`

Room 3 requires `@ConstructedBy` by default (only an experimental warning
suppression exists), but every documented declaration is the KMP
`expect object` form ‚Äî illegal in this single-platform module. This file
therefore ships without it, and the gate's KSP run gives the authoritative
answer:

- **If KSP compiles clean:** reflection-based instantiation still serves
  single-platform Android; done, no correction needed.
- **If KSP errors demanding `@ConstructedBy`:** paste the exact error. The
  prepared fix is a plain object ‚Äî
  `object CatsCarsDatabaseConstructor : RoomDatabaseConstructor<CatsCarsDatabase> { override fun initialize() = CatsCarsDatabase_Impl() }`
  ‚Äî plus `@ConstructedBy(CatsCarsDatabaseConstructor::class)` on the class,
  folded into this step per the no-fix-of-fix rule.

## Also queued by this step (before the Koin builder)

Room 3 **requires an explicit `SQLiteDriver`** at build time. The catalog
has no `androidx.sqlite` artifact yet ‚Äî the next two steps add
`androidx.sqlite:sqlite-bundled` (bundled driver: deterministic SQLite
version across devices, same blueprint reasoning as dynamic-color-off) to
the catalog and `app/build.gradle.kts`, then the Koin module builds the
database with `.setDriver(BundledSQLiteDriver())`.

package com.icodeforyou.catscarscoins.db

import androidx.room3.Database
import androidx.room3.RoomDatabase
import com.icodeforyou.catscarscoins.coins.data.CoinDao
import com.icodeforyou.catscarscoins.coins.data.CoinEntity

/**
 * The one app database (spec ¬ß5: single CatsCarsDatabase). Version 1
 * carries only the coins table; Cats (FTS4) and Cars entities join in
 * their phases with schema version bumps ‚Äî the exported schemas/ history
 * (spec 0.7) records each step.
 *
 * Room 3: androidx.room3 package; instantiation happens in the Koin module
 * via Room.databaseBuilder with an explicit SQLiteDriver (required in
 * Room 3 ‚Äî no driver, no database).
 */
@Database(
    entities = [CoinEntity::class],
    version = 1,
    exportSchema = true,
)
abstract class CatsCarsDatabase : RoomDatabase() {

    abstract fun coinDao(): CoinDao
}
