## Context

With the `RoomDbViewerRepository` verified and compiling clean, we must wire it into the app dependency graph via Koin. This step delivers a discrete `dbViewerModule` and reissues the central `AppModules` list to register the new vertical.

## Design

1. **Singleton Binding.** The `RoomDbViewerRepository` is declared as a `single` provider, binding straight to the `DbViewerRepository` domain interface. It extracts its `CatsCarsDatabase` dependency directly from the ambient graph.
2. **Modular App Architecture.** The reissued `AppModules.kt` appends `dbViewerModule` to the tail of the production dependency token collection list, maintaining the established load hierarchy perfectly.

## Verification gate

1. `./gradlew assembleDebug`
2. Expect: Compiles clean; dependency resolution structure is structurally verified.

## Complete file: `DbViewerModule.kt`

```kotlin
// di/DbViewerModule.kt
// CatsCarsCoins — spec 24.5.4. Complete file.
package com.icodeforyou.catscarscoins.di

import com.icodeforyou.catscarscoins.db.CatsCarsDatabase
import com.icodeforyou.catscarscoins.dbviewer.data.RoomDbViewerRepository
import com.icodeforyou.catscarscoins.dbviewer.domain.DbViewerRepository
import org.koin.core.module.Module
import org.koin.dsl.module

/**
 * Dependency injection graph slice for the Phase 5 Database Viewer feature.
 * Binds the Room-backed schema reader to the domain repository interface port.
 */
val dbViewerModule: Module = module {
    single<DbViewerRepository> {
        RoomDbViewerRepository(database = get<CatsCarsDatabase>())
    }
}

```
