## 24.0.3-05 — `gradle/libs.versions.toml` (KSP 2.3.3)

Replace the entire contents of `gradle/libs.versions.toml` (**Android view:** Gradle Scripts → `libs.versions.toml (Version Catalog)`. **Project view:** `CatsCarsCoins/gradle/libs.versions.toml`) with the file below. **Do not sync** — one sync at the end of 24.0.3-06.

**Why:** the post-sync warnings "Using kotlin.sourceSets DSL to add Kotlin sources is not allowed with built-in Kotlin" are KSP issue #2729 — the Kotlin-matched KSP line (`2.2.10-2.0.2`) registers generated sources the pre-AGP-9 way. KSP is Kotlin-version-independent since its 2.3.0 line; 2.3.1 added AGP 9 built-in Kotlin support and 2.3.3 completed the migration off the deprecated KGP API. Only change from 24.0.3-03: `ksp = "2.3.3"` and its comment. Do not use the `android.disallowKotlinSourceSets=false` suppression flag — that hides the cause instead of fixing it. If the IDE later offers a newer 2.3.x patch, it is safe to take.
