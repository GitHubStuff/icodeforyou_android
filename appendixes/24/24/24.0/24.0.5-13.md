## 24.0.5-13 — `values-v31/themes.xml` (OS splash: plain black, no icon)

> Supersedes the withdrawn inset-drawable version of this step. If `drawable/splash_logo_inset.xml` was created, delete it — nothing references it.

Replace the entire contents of `app/src/main/res/values-v31/themes.xml` with the file below. One change from 24.0.4-04: the `windowSplashScreenAnimatedIcon` line is **removed**.

**Why:** the API 31+ splash *icon slot* is circle-masked by the OS with no opt-out — that was the circle/oval crop. The full-rectangle technique never uses that slot: the OS splash is plain black, and the logo renders full-size in the in-app splash entry (next step), where there is no mask and the display duration is app-controlled. Correction #14: the logo was wired into the masked slot in 24.0.4-04.

### The file

```xml
<?xml version="1.0" encoding="utf-8"?>
<!-- values-v31/themes.xml — platform SplashScreen configuration (spec 0.4).
     OS splash is PLAIN BLACK by design: the platform's icon slot circle-masks
     any artwork, so the logo is deliberately NOT set here — it renders
     full-rectangle in the in-app splash entry instead. windowBackground stays
     black so no flash appears between splash dismissal and Compose's first
     frame. -->
<resources>

    <style name="Theme.CatsCarsCoins" parent="android:Theme.Material.Light.NoActionBar">
        <item name="android:windowSplashScreenBackground">@color/black</item>
        <item name="android:windowBackground">@drawable/splash_black</item>
    </style>
</resources>
```

### Verify

```zsh
./gradlew assembleDebug
```

**Run check:** launch — OS splash is now solid black (no circle, no logo), then "CatsCarsCoins" text, then Main. The logo returns full-rectangle in the next step.
