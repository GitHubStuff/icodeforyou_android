## The test

One instrumented test that proves the 1→2 auto-migration on a real
database, four claims at once:

1. **Data survives:** a coin row inserted at v1 (raw SQL — DAOs only speak
   the latest schema) reads back intact after migration.
2. **Schema validates:** `runMigrationsAndValidate(TEST_DB, 2, true)`
   diffs the migrated database against the committed `2.json` — a missed
   table, column, or trigger fails here, not on a user's device.
3. **Cats tables exist** and start empty.
4. **The migrated tables are writable and the virtual table exists:** a
   row inserted into `cats` reads back, and `cats_fts` answers a query.

   *Scope correction, adjudicated on-device:* the original MATCH probe
   asserted FTS sync at migration scope — but Room installs the
   external-content sync triggers in its **open** path, not in migration
   DDL, so the helper's raw connection legitimately has no triggers and
   the probe failed on both devices while everything else passed. FTS
   sync behavior is owned and proven by `RoomCatsRepositoryTest` (nine
   green through a properly opened Room); the migration test asserts
   only what migration owns.

## Room 3 testing API — adjudicated by the compiler

The delivered-first constructor guess was wrong; the compile errors
spelled out the real API, folded in:

- Constructor: `MigrationTestHelper(instrumentation, file =
  targetContext.getDatabasePath(TEST_DB), driver = BundledSQLiteDriver(),
  databaseClass = CatsCarsDatabase::class)` — the database **file** is
  fixed at construction.
- `createDatabase(version)` and `runMigrationsAndValidate(version,
  migrations = ...)` are **suspend** and take no name (the file is the
  ctor's) and no `validateDroppedTables` flag — the test body runs under
  `runTest` (coroutines-test is on the androidTest classpath since
  24.2.10). Auto-migrations come from `databaseClass`; the default empty
  `migrations` list is correct here.
- Connections are `androidx.sqlite` types: `execSQL` extension for
  writes, `prepare(...).use { step()/getLong()/getText() }` for reads.

## Complete file: `DatabaseMigrationTest.kt`

```kotlin
// db/DatabaseMigrationTest.kt
// CatsCarsCoins — spec 24.3.6. Complete file. androidTest sources.
// Correction folded in — Room 3 MigrationTestHelper API (adjudicated by
// the compiler): the constructor takes the database *file*; createDatabase
// and runMigrationsAndValidate are suspend, take only the version (the
// file is fixed at construction), and there is no validateDroppedTables
// flag. Test body now runs under runTest.
// Second correction (adjudicated on-device): the FTS MATCH probe is out of
// migration scope — Room installs the external-content sync triggers in
// its OPEN path, not in migration DDL, so the helper's raw connection has
// the tables but no triggers. FTS sync behavior is proven by
// RoomCatsRepositoryTest through a properly opened Room; this test now
// asserts what migration owns: schema, data survival, tables writable,
// the virtual table present.
package com.icodeforyou.catscarscoins.db

import androidx.room3.testing.MigrationTestHelper
import androidx.sqlite.driver.bundled.BundledSQLiteDriver
import androidx.sqlite.execSQL
import androidx.test.ext.junit.runners.AndroidJUnit4
import androidx.test.platform.app.InstrumentationRegistry
import kotlinx.coroutines.test.runTest
import org.junit.Assert.assertEquals
import org.junit.Assert.assertTrue
import org.junit.Rule
import org.junit.Test
import org.junit.runner.RunWith

private const val TEST_DB = "migration-test.db"

@RunWith(AndroidJUnit4::class)
class DatabaseMigrationTest {

    private val instrumentation = InstrumentationRegistry.getInstrumentation()

    @get:Rule
    val helper = MigrationTestHelper(
        instrumentation = instrumentation,
        file = instrumentation.targetContext.getDatabasePath(TEST_DB),
        driver = BundledSQLiteDriver(),
        databaseClass = CatsCarsDatabase::class,
    )

    @Test
    fun migrate1To2_preservesCoinsAndCreatesCatsTables() = runTest {
        helper.createDatabase(1).apply {
            execSQL(
                "INSERT INTO coins (amount_cents, recorded_at) VALUES (434317, 1000)",
            )
            close()
        }

        val migrated = helper.runMigrationsAndValidate(2)

        migrated.prepare("SELECT amount_cents, recorded_at FROM coins").use { statement ->
            assertTrue(statement.step())
            assertEquals(434317L, statement.getLong(0))
            assertEquals(1000L, statement.getLong(1))
            assertTrue(!statement.step())
        }

        migrated.prepare("SELECT COUNT(*) FROM cats").use { statement ->
            assertTrue(statement.step())
            assertEquals(0L, statement.getLong(0))
        }

        migrated.execSQL(
            "INSERT INTO cats (id, image_url, breed_id, breed_name, origin, temperament, life_span, description) " +
                "VALUES ('abc', 'https://x/y.jpg', 'beng', 'Bengal', 'United States', 'Alert, Agile', '12 - 15', 'Bengals are a lot of fun')",
        )

        migrated.prepare("SELECT breed_name FROM cats WHERE id = 'abc'").use { statement ->
            assertTrue(statement.step())
            assertEquals("Bengal", statement.getText(0))
        }

        migrated.prepare("SELECT COUNT(*) FROM cats_fts").use { statement ->
            assertTrue(statement.step())
        }

        migrated.close()
    }
}
```
