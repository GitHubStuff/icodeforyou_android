## Changes

- **Added** `applicationScope` — `SupervisorJob() + Dispatchers.Default`.
  Process-lifetime by design: it dies with the process, so no cancellation
  path exists or is needed; Default because nothing in the feed touches the
  main thread.
- **Added** `startPreferencesSnapshotFeed()`, called immediately after
  `startKoin`: resolves `PreferencesRepository` (via Koin's
  `ComponentCallbacks.get()`) and collects `preferences` into
  `PreferencesSnapshot.update`. This is the **single production writer**
  the 24.1.15 discipline requires. The collect body is a lambda — same
  Kotlin limitation as 24.1.8: references don't reliably adapt to suspend
  functional types.
- **KDoc updated:** MyApp's responsibility is stated as bootstrap (Koin +
  snapshot feed), keeping the SRP claim honest.
- Koin startup block itself unchanged.

## Ordering guarantee

The feed starts in `Application.onCreate`, before any Activity exists —
`PreferencesSnapshot` is warm before the first composable could ever call
`hapticsEnabled()`. An empty store emits `DEFAULTS`, which equals the
snapshot's initial value, so even the pre-first-emission window reads
correctly.

## Complete file: `MyApp.kt`

```kotlin
// MyApp.kt
// CatsCarsCoins — spec 24.1.16. Complete file.
// Change from previous version: after startKoin, an application-scoped
// collector feeds PreferencesSnapshot from the repository flow — the single
// production writer behind the spec §17 synchronous accessors.
package com.icodeforyou.catscarscoins

import android.app.Application
import com.icodeforyou.catscarscoins.di.appModules
import com.icodeforyou.catscarscoins.preferences.PreferencesSnapshot
import com.icodeforyou.catscarscoins.preferences.domain.PreferencesRepository
import kotlinx.coroutines.CoroutineScope
import kotlinx.coroutines.Dispatchers
import kotlinx.coroutines.SupervisorJob
import kotlinx.coroutines.launch
import org.koin.android.ext.android.get
import org.koin.android.ext.koin.androidContext
import org.koin.android.ext.koin.androidLogger
import org.koin.core.context.startKoin

/**
 * Application entry point (spec 0.5). Bootstrap responsibilities only:
 * start Koin with the aggregated feature modules, then start the
 * preferences snapshot feed that backs the spec §17 accessors
 * (hapticsEnabled() / buttonFontSize() read synchronously; something must
 * keep the synchronous source current).
 */
class MyApp : Application() {

    /**
     * Process-lifetime scope: SupervisorJob so a failed child never kills
     * the feed's siblings, Default dispatcher — nothing here touches the
     * main thread. Never cancelled; it dies with the process.
     */
    private val applicationScope = CoroutineScope(SupervisorJob() + Dispatchers.Default)

    override fun onCreate() {
        super.onCreate()
        startKoin {
            androidLogger()
            androidContext(this@MyApp)
            modules(appModules)
        }
        startPreferencesSnapshotFeed()
    }

    /**
     * The single production writer of [PreferencesSnapshot]: every
     * repository emission — including the DEFAULTS emitted by an empty
     * store — lands in the snapshot, so accessor reads are current from
     * first launch on.
     */
    private fun startPreferencesSnapshotFeed() {
        val repository = get<PreferencesRepository>()
        applicationScope.launch {
            repository.preferences.collect { preferences ->
                PreferencesSnapshot.update(preferences)
            }
        }
    }
}
```
