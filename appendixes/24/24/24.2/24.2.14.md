## Design

- **Wire shape verbatim.** The endpoint wraps the payload in a `data`
  envelope; two `@Serializable` classes mirror it. All three payload
  fields (`amount`, `base`, `currency`) are Strings on the wire and stay
  Strings here — a DTO records the wire, it does not interpret it.
- **Explicit `@SerialName` on every property** even where names match:
  wire names survive any Kotlin rename, the same single-source discipline
  as `@ColumnInfo` (24.2.3).
- **The amount conversion is explicitly out of scope** — the KDoc points
  at the mapper (next steps), which parses `String → Long` cents via
  `BigDecimal`, never `Double`. Nothing downstream of the mapper ever
  sees the string.
- `remote/` subpackage under `coins/data/` — wire types quarantined from
  Room types.

// coins/data/remote/CoinbaseSpotPriceDto.kt

package com.icodeforyou.catscarscoins.coins.data.remote

import kotlinx.serialization.SerialName
import kotlinx.serialization.Serializable

/**
 * Coinbase spot-price wire shape (GET /v2/prices/BTC-USD/spot):
 * ```
 * { "data": { "amount": "4343.17", "base": "BTC", "currency": "USD" } }
 * ```
 * Every payload field is a String on the wire and stays a String here —
 * DTOs mirror the wire verbatim, no interpretation. The amount's
 * String → Long-cents conversion is the mapper's job (next steps), never
 * the DTO's, and never via Double (spec Decision Log: exact cents).
 */
@Serializable
data class CoinbaseSpotPriceResponseDto(

    @SerialName("data")
    val data: CoinbaseSpotPriceDto,
)

@Serializable
data class CoinbaseSpotPriceDto(

    @SerialName("amount")
    val amount: String,

    @SerialName("base")
    val base: String,

    @SerialName("currency")
    val currency: String,
)
```
