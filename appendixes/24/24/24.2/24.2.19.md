## Changes

- **Added** `single<CoinbaseApi>`: Retrofit built on `CoinbaseApi.BASE_URL`,
  the shared `OkHttpClient` and `Json` (from `networkModule`), and the
  first-party kotlinx converter
  (`json.asConverterFactory("application/json".toMediaType())`). The
  Retrofit instance is local to the definition — only the API interface is
  a Koin single, because nothing else needs the coins Retrofit.
- **Added** `JSON_MEDIA_TYPE` file constant — no magic strings.
- `CoinsRepository` binding unchanged from 24.2.12.

## Correction folded in — converter package

The first-party converter lives at
`retrofit2.converter.kotlinx.serialization.asConverterFactory` (dotted
package). The initially delivered import used `kotlinxserialization`
(undotted) and failed to resolve, taking `asConverterFactory` down with it
(the IDE's "file types are usually capitalized" hints were noise from the
unresolved symbol). Import corrected; no other change.

## Complete file: `CoinsModule.kt`

```kotlin
// coins/di/CoinsModule.kt
// CatsCarsCoins — spec 24.2.19. Complete file.
// Change from 24.2.12: CoinbaseApi single added (Retrofit on the shared
// OkHttpClient + Json from networkModule).
// Correction folded in: the first-party converter's package is
// retrofit2.converter.kotlinx.serialization (dotted), not
// retrofit2.converter.kotlinxserialization.
package com.icodeforyou.catscarscoins.coins.di

import com.icodeforyou.catscarscoins.coins.data.RoomCoinsRepository
import com.icodeforyou.catscarscoins.coins.data.remote.CoinbaseApi
import com.icodeforyou.catscarscoins.coins.domain.CoinsRepository
import kotlinx.serialization.json.Json
import okhttp3.MediaType.Companion.toMediaType
import okhttp3.OkHttpClient
import org.koin.dsl.module
import retrofit2.Retrofit
import retrofit2.converter.kotlinx.serialization.asConverterFactory

/** Converter media type — appears exactly once. */
private const val JSON_MEDIA_TYPE = "application/json"

/**
 * Coins feature wiring (spec: Koin runtime DSL). Consumers inject the
 * domain contract and the API interface; Room and Retrofit types stay
 * behind this file (DIP). CoinDao resolves from databaseModule,
 * OkHttpClient and Json from networkModule.
 */
val coinsModule = module {

    single<CoinsRepository> {
        RoomCoinsRepository(coinDao = get())
    }

    single<CoinbaseApi> {
        Retrofit.Builder()
            .baseUrl(CoinbaseApi.BASE_URL)
            .client(get<OkHttpClient>())
            .addConverterFactory(
                get<Json>().asConverterFactory(JSON_MEDIA_TYPE.toMediaType()),
            )
            .build()
            .create(CoinbaseApi::class.java)
    }
}
```
