## Design

- **One method:** `suspend fun currentAmountCents(): Long`. The engine's
  entire knowledge of the outside world is "give me the price in exact
  cents."
- **Why a port instead of the engine calling `CoinbaseApi` directly:** the
  engine is the phase's core logic (interval, pause, dedup, refresh) and
  must be testable on virtual time with a hand fake. Wire concerns —
  envelope unwrap, `toUsdCents` — belong to the adapter
  (`CoinbaseCoinPriceSource`, a later step). Hexagonal at its smallest
  useful size; no speculative parameters (pair is fixed per spec).
- **Failure contract stated on the interface:** throw on failure; the
  engine skips the tick and keeps polling. Written where implementers and
  the engine's tests both read it.

```kotlin
// coins/domain/CoinPriceSource.kt
// CatsCarsCoins — spec 24.2.21. Complete file.
package com.icodeforyou.catscarscoins.coins.domain

/**
 * Domain port for "what does BTC-USD cost right now, in exact cents."
 *
 * The polling engine depends on this, never on Retrofit or wire types —
 * which is what lets the engine's tests (next step) drive it with a hand
 * fake and virtual time. The data-layer adapter (Coinbase API → DTO →
 * toUsdCents) implements it.
 *
 * Failure contract: implementations throw on network or parse failure;
 * the engine treats a thrown tick as "no sample this tick" and keeps
 * polling.
 */
interface CoinPriceSource {

    suspend fun currentAmountCents(): Long
}
```
