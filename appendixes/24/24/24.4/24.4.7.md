## Design (parallel to `CatsRemoteSource`)

1. **One method, domain types out** — `fetchPage(page): List<Manufacturer>`.
   The adapter (vPIC + DTO mapping, later step) owns all wire knowledge;
   the repository composes port + DAO.
2. **Why:** the repository's value is its FTS query policy and
   upsert-by-id semantics — tested instrumented on real SQLite with this
   port faked, the same seam that made `RoomCatsRepository` testable.
3. **`page`** parameter (vPIC pagination), not `limit` — vPIC pages are
   server-sized; the repository passes `FETCH_PAGE`.
4. Failure contract on the interface: throw; the repository propagates.
