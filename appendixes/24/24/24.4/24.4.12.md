# 24.4.12 — `CarApi` (Retrofit interface)

## Objective

- Declare the single vPIC endpoint the Cars vertical consumes.
- Static portion of the URL carries `format=json`; the page number is the only dynamic input.
- Declaration only — no logic, no dedicated test. Behavior is already pinned by `CarDtoTest` (decoding/mapping) and `RoomCarsRepositoryTest` (persistence); the thin adapter over this interface lands next (24.4.13).

## Design notes

- Relative path `vehicles/GetAllManufacturers?format=json` — **no leading slash**, so it resolves against the `carsModule` base URL `https://vpic.nhtsa.dot.gov/api/` (module lands in 24.4.14).
- Retrofit appends `@Query("page")` to the static query string, producing `...?format=json&page=N`.
- Return type is `ManufacturersResponseDto` — the shared kotlinx-serialization converter decodes the envelope; rows flow onward via `mapNotNull { it.toDomainOrNull() }` in the remote source.
- Keyless API — this interface carries no auth annotations, and `carsModule` will carry **no interceptor** (contrast with `catsModule`'s x-api-key interceptor).
- Suspend fun — Retrofit 3 native coroutine support, no wrapper types.
