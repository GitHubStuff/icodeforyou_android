## Design (parallel to `CatDao`)

1. **Four members, each a contract clause:** blank-query observation
   (`observeAll`), FTS search (`observeMatching` — content joined to
   shadow on `rowid`), `@Upsert` by PK (`Mfr_ID`), `deleteAll` (the FTS
   shadow follows via Room's generated triggers).
2. **One ordering, twice:** `name ASC, id ASC` in both queries — search
   and full list can never disagree.
3. **Raw MATCH parameter:** the repository owns text→FTS-syntax
   translation; the DAO stays dumb SQL.

```kotlin
// cars/data/CarDao.kt
// CatsCarsCoins — spec 24.4.4. Complete file.
package com.icodeforyou.catscarscoins.cars.data

import androidx.room3.Dao
import androidx.room3.Query
import androidx.room3.Upsert
import kotlinx.coroutines.flow.Flow

/**
 * Room 3 DAO for the manufacturers table and its FTS4 shadow (spec Phase
 * 4). Both observations share one deterministic ordering — name
 * alphabetical, id tie-break — so the full list and a search result can
 * never disagree about order. Parallel to CatDao.
 *
 * [observeMatching] takes a raw FTS MATCH expression; translating a
 * user's search text into FTS syntax is repository policy, not SQL's job.
 */
@Dao
interface CarDao {

    @Query("SELECT * FROM manufacturers ORDER BY name ASC, id ASC")
    fun observeAll(): Flow<List<ManufacturerEntity>>

    @Query(
        """
        SELECT manufacturers.* FROM manufacturers
        JOIN manufacturers_fts ON manufacturers.rowid = manufacturers_fts.rowid
        WHERE manufacturers_fts MATCH :match
        ORDER BY name ASC, id ASC
        """,
    )
    fun observeMatching(match: String): Flow<List<ManufacturerEntity>>

    /**
     * Insert-or-update by primary key (Mfr_ID) — the 24.4.2 upsert
     * contract as a single Room 3 annotation.
     */
    @Upsert
    suspend fun upsertAll(entities: List<ManufacturerEntity>)

    /** Reset App hook (spec §16). The FTS shadow follows via triggers. */
    @Query("DELETE FROM manufacturers")
    suspend fun deleteAll()
}
```
