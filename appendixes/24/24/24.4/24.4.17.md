# 24.4.17 — `CarsScreen` + `CarsModule` reissue (VM registered)

## Objective

- Cars destination UI on the shared `SwipeableRefreshableList` capstone — second reuse after Cats/Coins.
- Two files, **must land together**: the screen resolves `CarsViewModel` via `koinViewModel()`, so the module reissue registering `viewModelOf(::CarsViewModel)` is a compile-time-fine but runtime-fatal dependency.

## `CarsScreen` design (mirror of `CatsScreen`, deltas listed)

- Same skeleton: search `OutlinedTextField` + `AppButton` Refresh row, then `SwipeableRefreshableList` weighted to fill.
- Same failure plumbing: `refreshFailures` collected in `LaunchedEffect(Unit)` → `notifier.show(READ_ONLY_TOAST_MILLIS)` — signaled by the VM, presented by the screen.
- Same haptics/font wiring on `AppButton`: `hapticsEnabled()` + `buttonFontSize()` accessors.
- Deltas from Cats:
  - **No images** — vPIC has none, so rows are two-line text (name titleMedium, country bodyMedium single-line ellipsized).
  - **No row tap** — manufacturer detail is spec-optional (§8.5) and skipped; no `onManufacturerSelected` seam, no `clickable`.
  - Empty states: blank query → "No manufacturers yet / Tap Refresh to fetch a page from NHTSA vPIC."; non-blank → no-match with the query echoed.
  - `itemContent` owns its `HorizontalDivider`, per the capstone's no-divider-policy contract.
- Stateless below the ViewModel seam; `CarsContent` takes values + callbacks only.
