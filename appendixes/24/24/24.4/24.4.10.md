# 24.4.10 — `CarDtoTest` (Cars wire layer, RED)

## Objective

- Open the Cars wire layer with a failing JVM test, parallel to `CatDtoTest`.
- Test decodes **literal vPIC JSON** (envelope + rows) and exercises the `toDomainOrNull` mapping contract.
- Red gate: unresolved `CarDto` / `ManufacturersResponseDto` (both land in one file, `CarDto.kt`, at the green step — wire types for one endpoint are cohesive).

## Verified vPIC wire shape (checked 2026-07-06 against live vPIC docs)

- Base URL: `https://vpic.nhtsa.dot.gov/api/`
- Endpoint: `vehicles/GetAllManufacturers?format=json&page=N`
- Keyless. No API key, no interceptor in `carsModule`.
- Paged: 100 rows per page.
- Envelope keys: `Count`, `Message`, `SearchCriteria`, `Results`.
- Row keys we consume:
  - `Mfr_ID` — Int
  - `Mfr_Name` — String
  - `Country` — String, **sometimes null**
- Row keys we ignore (must not break decoding): `Mfr_CommonName`, `VehicleTypes[]` (nested objects), others.

Representative row (verbatim shape from live dataset):

```json
{
  "Country": "UNITED STATES (USA)",
  "Mfr_CommonName": "Tesla",
  "Mfr_ID": 955,
  "Mfr_Name": "TESLA, INC.",
  "VehicleTypes": [ { "IsPrimary": true, "Name": "Passenger Car" } ]
}
```

## Mapping contract under test

- `ManufacturersResponseDto` declares only `Results` (`ignoreUnknownKeys` handles the rest of the envelope).
- `CarDto` fields all nullable with `null` defaults (defensive wire posture): `mfrId`, `mfrName`, `country`.
- `CarDto.toDomainOrNull(): Manufacturer?`
  - `mfrId == null` → `null`
  - `mfrName` null or blank → `null`
  - `country` null or blank → domain `country = ""` (blank-country default)
