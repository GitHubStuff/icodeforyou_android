# 24.4.11 — `CarDto.kt` (Cars wire layer, GREEN)

## Objective

- Resolve the 24.4.10 red gate with one file containing both wire types for the `GetAllManufacturers` endpoint:
  - `ManufacturersResponseDto` — envelope, declares only `Results` (`ignoreUnknownKeys` absorbs `Count`, `Message`, `SearchCriteria`).
  - `CarDto` — row, all fields nullable with `null` defaults (defensive wire posture; vPIC ships nulls and extra keys).
- `toDomainOrNull()` mapping rules:
  - `mfrId == null` → row rejected (`null`).
  - `mfrName` null or blank → row rejected (`null`).
  - `country` null or blank → domain `country = ""` (blank-country default, named constant, no magic literal at the decision site).

## Design notes

- `@SerialName` pins the exact vPIC keys (`Mfr_ID`, `Mfr_Name`, `Country`) so property names stay idiomatic Kotlin.
- Unknown keys (`Mfr_CommonName`, `VehicleTypes[]`) never appear in the DTO — the shared `Json { ignoreUnknownKeys = true }` drops them.
- Mapper is an extension returning `Manufacturer?` so the remote source composes with `mapNotNull` (lands in 24.4.13).
