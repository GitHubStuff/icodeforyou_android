## Room 3 note

Room 3.0 lives in a **new package** — imports are `androidx.room3.Entity` /
`PrimaryKey` / `ColumnInfo`, matching the project's locked
`androidx.room3` coordinates. Typing the 2.x `androidx.room.*` imports out
of muscle memory produces unresolved references against this project's
dependencies.

## Design

- **Wire names per spec:** table `coins`, columns `amount_cents`,
  `recorded_at` — explicit `@ColumnInfo`, never derived from property
  names, so a rename refactor can't silently change the schema.
- **PK convention:** `id`, auto-generated; default `Coin.UNSAVED_ID` (the
  named sentinel from 24.2.1) lets inserts omit it.
- **`toDomain()` mapper** lives beside the shape it maps; there is no
  `fromDomain` — the repository constructs entities from `record(...)`'s
  primitives, so no caller can smuggle in an id.
- **CREATE TABLE equivalent in the KDoc** per the spec's database-section
  requirement: the SQL the annotation produces is visible at the source.

// coins/data/CoinEntity.kt

package com.icodeforyou.catscarscoins.coins.data

import androidx.room3.ColumnInfo
import androidx.room3.Entity
import androidx.room3.PrimaryKey
import com.icodeforyou.catscarscoins.coins.domain.Coin

/**
 * Room row for one BTC-USD price sample (spec: coins table
 * { id, amount_cents, recorded_at }). Room 3 — androidx.room3 imports.
 *
 * CREATE TABLE equivalent:
 * ```
 * CREATE TABLE coins (
 *     id           INTEGER PRIMARY KEY AUTOINCREMENT NOT NULL,
 *     amount_cents INTEGER NOT NULL,
 *     recorded_at  INTEGER NOT NULL
 * )
 * ```
 *
 * Data-layer type only: everything above the repository sees [Coin].
 */
@Entity(tableName = "coins")
data class CoinEntity(

    @PrimaryKey(autoGenerate = true)
    val id: Long = Coin.UNSAVED_ID,

    @ColumnInfo(name = "amount_cents")
    val amountCents: Long,

    @ColumnInfo(name = "recorded_at")
    val recordedAt: Long,
)

fun CoinEntity.toDomain(): Coin = Coin(
    id = id,
    amountCents = amountCents,
    recordedAtEpochMillis = recordedAt,
)
```
