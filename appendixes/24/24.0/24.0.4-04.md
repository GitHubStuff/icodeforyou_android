## 24.0.4-04 — Themes (base + values-v31 splash)

Two files (spec 0.4: `values/themes.xml` + `values-v31/themes.xml`, OS splash → black). With minSdk 31 the v31 qualifier is technically always active — it is kept because the spec's file tree pins it, and it keeps splash attributes isolated from the base theme.

### File 1 — replace `app/src/main/res/values/themes.xml`

```xml
<?xml version="1.0" encoding="utf-8"?>
<!-- themes.xml — base app theme shell. Visual theming lives in Compose
     (AppTheme, spec 0.5); this XML theme only hosts window-level attributes. -->
<resources>

    <style name="Theme.CatsCarsCoins" parent="android:Theme.Material.Light.NoActionBar" />
</resources>
```

(Unchanged from wizard output except the comment — listed for the record.)

### File 2 — create `app/src/main/res/values-v31/themes.xml`

**Project view:** right-click `res` → New → Android Resource File → File name `themes`, Resource type `Values`, add qualifier **Version** = 31. Full content:

```xml
<?xml version="1.0" encoding="utf-8"?>
<!-- values-v31/themes.xml — platform SplashScreen configuration (spec 0.4).
     OS splash renders splash_logo on a black background; windowBackground is
     the black drawable so no white flash appears between splash dismissal and
     Compose's first frame. -->
<resources>

    <style name="Theme.CatsCarsCoins" parent="android:Theme.Material.Light.NoActionBar">
        <item name="android:windowSplashScreenBackground">@color/black</item>
        <item name="android:windowSplashScreenAnimatedIcon">@drawable/splash_logo</item>
        <item name="android:windowBackground">@drawable/splash_black</item>
    </style>
</resources>
```

Notes: `@color/black` already exists in the wizard's `colors.xml`; `splash_logo` and `splash_black` were created in 24.0.4-02. `windowSplashScreenBackground` accepts colors only — that is why the black exists twice (color for the splash attribute, drawable for the window background).

### Verify

**Build → Make Project** — green. Both theme files show no unresolved-resource highlights.
