## 24.0.5-06 — `nav/AppNavKeys.kt`

Create new file `app/src/main/java/com/icodeforyou/catscarscoins/nav/AppNavKeys.kt` (**Project view:** right-click `java/com.icodeforyou.catscarscoins` → New → Package → `nav`, then New → Kotlin Class/File → File → `AppNavKeys`). Full content:

```kotlin
// nav/AppNavKeys.kt
package com.icodeforyou.catscarscoins.nav

import androidx.navigation3.runtime.NavKey
import kotlinx.serialization.Serializable

/**
 * App-level Nav3 keys (spec 0.5: NavDisplay Splash -> Main -> empty stubs).
 * Keys are @Serializable so the developer-owned back stack survives process
 * death (spec §5: kotlinx.serialization also serializes NavKeys). Feature
 * keys live in each feature's own nav/ package — this sealed interface only
 * covers app-shell destinations.
 */
sealed interface AppNavKey : NavKey

@Serializable
data object SplashKey : AppNavKey

@Serializable
data object MainKey : AppNavKey
```

**Why:** the keys land before the `NavDisplay` that consumes them — same forward-reference discipline as `appModules` before `MyApp`. This is also the first file exercising the kotlinx.serialization compiler plugin; a green build here proves that whole plugin chain.
