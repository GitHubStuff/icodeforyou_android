## `FakeCarsRepository` design (mirror of `FakeCatsRepository`)

- Test source set, contract's package (`cars/domain`) — the established Fake pattern.
- `state` exposed for scenario scripting; `queriesSeen` records every query (debounce assertions).
- Filter stand-in covers **both FTS-indexed fields** (24.4.1: name, country) via case-insensitive contains — real MATCH behavior stays owned by `RoomCarsRepositoryTest`.
- `refreshCount`, `failNextRefresh` (fails once, `IOException`), optional `refreshGate` (`CompletableDeferred`) to hold a refresh in-flight.

## `CarsViewModelTest` coverage (10 tests)

- Blank query shows everything after the debounce window.
- Query change narrows the results (`"tes"` → Tesla).
- Query matches the country field (`"germany"` → BMW) — Cars-specific, both indexed fields exercised at the VM seam.
- Rapid typing subscribes only the final query (`queriesSeen == ["", "tes"]`).
- `onRefresh` delegates to the repository.
- Refresh failure emits one failure event; refresh success emits none.
- `isRefreshing` starts false; true in-flight → false after success (gated); clears after a failed refresh (gated).

## Pinned `CarsViewModel` contract (green lands in 24.4.16)

- `query: StateFlow<String>`, `onQueryChanged(value)`.
- `manufacturers: StateFlow<List<Manufacturer>>` = query → `debounce(QUERY_DEBOUNCE_MILLIS)` → `flatMapLatest { repository.manufacturers(it) }` → `stateIn(WhileSubscribed(SUBSCRIPTION_STOP_TIMEOUT_MS), emptyList())`.
- `refreshFailures: SharedFlow<Unit>` — signaled, never presented (screen owns the 750 ms read-only toast).
- `isRefreshing: StateFlow<Boolean>` — flipped in try/finally.
- `companion { const val QUERY_DEBOUNCE_MILLIS: Long = 300L }`.
