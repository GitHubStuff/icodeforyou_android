## Context

Phase 1 builds the Preferences core — the shared dependency every later
feature consumes (theme drives AppTheme, polling drives Coins, haptics and
the accessors drive every AppButton). Clean Architecture sets the in-phase
order: domain contract → tests → DataStore implementation → Koin module →
wire into `appModules`. This step is the domain contract's model.

## Locked spec facts encoded

- Four keys: theme, polling interval (5–30, `Int`), polling pause, haptics.
- Defaults / reset state (spec §16 step 2): theme=dark, haptics=on,
  polling=5, pause=true.
- `buttonFontSize()` is a design-system global, **not** a preference — it is
  deliberately absent from this model.
- Theme `SYSTEM` resolves via `isSystemInDarkTheme()` at composition time —
  resolution is a UI concern and does not live in this layer.

## Design

- Pure Kotlin: zero Android, DataStore, or serialization imports. The data
  layer maps this type to and from DataStore keys; domain and UI layers see
  only this type (Dependency Inversion).
- Polling bounds are named constants plus an `IntRange`
  (`POLLING_INTERVAL_RANGE`) — no magic numbers; the Settings slider and the
  data-layer clamp both read the same single source of truth.
- `DEFAULTS` is one canonical instance serving first-run and
  `ResetAppUseCase.resetToDefaults()` — the reset behavior and first-run
  behavior cannot drift apart.

```kotlin
// preferences/domain/AppPreferences.kt
// CatsCarsCoins — spec 24.1.1. Complete file.
package com.icodeforyou.catscarscoins.preferences.domain

/**
 * Theme selection (spec §18): two explicit palettes plus a system-following
 * mode. SYSTEM resolves at composition time via isSystemInDarkTheme() —
 * resolution is a UI concern and deliberately absent from this layer.
 */
enum class ThemeMode {
    LIGHT,
    DARK,
    SYSTEM,
}

/**
 * The four persisted preferences (spec §Decision Log: "theme, polling
 * (5–30 int), pause, haptics"). Pure domain model — no Android, no
 * DataStore, no serialization knowledge. The data layer maps this to and
 * from Preferences DataStore keys; everything above consumes only this type.
 */
data class AppPreferences(
    val themeMode: ThemeMode,
    val pollingIntervalSeconds: Int,
    val pollingPaused: Boolean,
    val hapticsEnabled: Boolean,
) {
    companion object {

        /** Inclusive polling bounds (spec: 5–30 seconds, integer). */
        const val POLLING_INTERVAL_MIN_SECONDS: Int = 5
        const val POLLING_INTERVAL_MAX_SECONDS: Int = 30

        val POLLING_INTERVAL_RANGE: IntRange =
            POLLING_INTERVAL_MIN_SECONDS..POLLING_INTERVAL_MAX_SECONDS

        /**
         * Spec reset/default state (spec §16 Reset App order step 2):
         * theme=dark, haptics=on, polling=5, pause=true. Used both for
         * first-run values and by ResetAppUseCase's resetToDefaults().
         */
        val DEFAULTS: AppPreferences = AppPreferences(
            themeMode = ThemeMode.DARK,
            pollingIntervalSeconds = POLLING_INTERVAL_MIN_SECONDS,
            pollingPaused = true,
            hapticsEnabled = true,
        )
    }
}
```
