## Context

`app/build.gradle.kts` already configures the Room 3 Gradle plugin with
`schemaDirectory("$projectDir/schemas")`. Room only writes into that
directory when the first `@Database` compiles — which hasn't happened yet —
so the directory doesn't exist and therefore isn't tracked. Git tracks
files, not directories; an empty directory needs a placeholder file.

Both `.gitignore` files were verified: nothing excludes `schemas/`.

## Commands

From the repo root:

```
mkdir -p app/schemas
touch app/schemas/.gitkeep
git add app/schemas/.gitkeep
git commit -m "Phase 0.7: track Room schema directory (spec 24.0.7)"
```

## Why schemas are tracked

Room exports one JSON file per database version into `schemas/`. Tracked
schemas are the input to Room's migration testing (`room3-testing`, already
pinned) and the historical record that makes destructive-migration
regressions visible in code review. Untracked schemas silently defeat both.

## Pass criteria

- `git status` clean after the commit.
- `app/schemas/.gitkeep` present in the repository.
- Later, when the first `@Database` compiles, `1.json` appears next to
  `.gitkeep` and gets committed with the database code.
